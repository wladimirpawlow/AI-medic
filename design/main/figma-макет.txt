AI-medic — Figma-макет v1

1. Общие настройки Figma
- Страницы:
  - Page: "Desktop"
- Базовый фрейм приложения:
  - Frame: "App / Shell" (1440×900, Desktop)
  - Layout Grid: Columns 12, margin 80, gutter 24
  - Фон: #F5F7FA
  - Базовый текст: Inter / Roboto 14 px, цвет #111827
  - Акцентный цвет: #2563EB (кнопки, активные элементы)
  - Вторичный цвет: #1E40AF (ховеры/бордеры)
  - Нейтральные:
    - Текст вторичный: #6B7280
    - Бордер: #E5E7EB
    - Фон таблицы/чипов: #F3F4F6

2. Общие компоненты
- Component: "Topbar"
  - Высота: 56
  - Фон: #FFFFFF, тень: 0,1,2,0.05
  - Содержимое:
    - Слева: логотип (текст "AI-medic", акцентный цвет)
    - Справа: имя пользователя + иконка профиля

- Component: "Navbar / Tabs"
  - Расположение: под Topbar, внутренняя ширина контейнера (по сетке)
  - Табы (Text 14 semi-bold):
    - "Осмотры" (active)
    - "Наборы"
    - "Запуски"
    - "Расчёты"
  - Активный таб:
    - Цвет текста: #111827
    - Нижний бордер: 2 px, #2563EB
  - Неактивные:
    - Цвет текста: #6B7280

- Component: "Page Header"
  - Заголовок (H1 20–24 px): слева
  - Справа: primary button и доп. кнопки

- Component: "Button / Primary"
  - Высота: 36
  - Радиус: 6
  - Фон: #2563EB
  - Текст: #FFFFFF, 14 semi-bold
  - Пример текста: "Загрузить осмотры", "Создать набор"

- Component: "Filter Bar"
  - Горизонтальный контейнер, высота ~56
  - Фон: #FFFFFF, бордер снизу #E5E7EB
  - Элементы:
    - Search input (плейсхолдер "Поиск…")
    - Date range (иконка календаря)
    - Dropdown(ы) по контексту экрана (см. ниже)

- Component: "Table"
  - Header row: фон #F9FAFB, текст #6B7280, 12–13 px uppercase/medium
  - Row: высота 44–48, бордер снизу #E5E7EB
  - Hover: фон #F3F4F6
  - Optional: чекбокс в первой колонке

- Component: "Tag / Status"
  - Высота: 20–22
  - Радиус: 999
  - Варианты:
    - "Новый" — фон #DBEAFE, текст #1D4ED8
    - "В обработке" — фон #FEF3C7, текст #92400E
    - "Готов" — фон #D1FAE5, текст #065F46
    - "Ошибка" — фон #FEE2E2, текст #991B1B

- Component: "Card"
  - Фон #FFFFFF, радиус 8, паддинги 16–20, тень слабая

3. Экран "Осмотры" (список)
- Frame: "Desktop / Осмотры"
  - Базовый слой: инстанс "App / Shell"
  - Topbar + Navbar: активен таб "Осмотры"
  - Page Header:
    - Title: "Осмотры"
    - Right:
      - Primary button: "Загрузить осмотры"
      - Secondary button (outline): "Создать набор из выбранных"
  - Filter Bar:
    - Search input: "Поиск по ID, пациенту, источнику"
    - Dropdown: "Источник" (Все / Источник 1 / …)
    - Dropdown: "Статус загрузки" (Все / Новый / Готов / Ошибка)
    - Date range: "Период медосмотра"
  - Контент:
    - Table: "Осмотры"
      - Колонки:
        - [Checkbox]
        - "ID осмотра"
        - "Дата осмотра"
        - "Пациент" (если ФИО не используется — "Идентификатор пациента")
        - "Источник" (внешний источник данных)
        - "Кол-во файлов"
        - "Статус загрузки" (Tag / Status)
        - "Входит в наборы" (число / список чипов-названий наборов)
        - "Действия" (иконка "глаз" / текст "Открыть")
    - Пагинация под таблицей (компонент "Pagination" — номера страниц + "Следующая")

7. Экран "Осмотр" (детальная страница)
- Frame: "Desktop / Осмотр — детали"
  - Topbar + Navbar: активен таб "Осмотры"
  - Верхняя часть:
    - Breadcrumb: "Осмотры / Осмотр #12345"
    - Page Header:
      - Title: "Осмотр #12345"
      - Subtitle (вторичный текст): "Дата осмотра: DD.MM.YYYY · Источник: <название>"
      - Actions (справа):
        - Primary button: "Запустить в САВ"
        - Secondary: "Добавить в набор"
  - Макет: двухколоночный
    - Левая колонка (~30–35% ширины):
      - Card "Общие данные":
        - Поля в 2 колонки или список:
          - "ID осмотра"
          - "Пациент" / "Идентификатор пациента"
          - "Дата осмотра"
          - "Источник"
          - "Наборы" (чипы-названия наборов)
      - Card "Исходные данные":
        - Список файлов (иконка видео/фото, название файла, длительность/размер)
    - Правая колонка (~65–70%):
      - Card с табами:
        - Tabs:
          - "Результаты анализа" (значения вероятностей из САВ)
          - "Решения" (ДА/НЕТ/Н.О. по замечаниям)
      - Вкладка "Результаты анализа":
        - Таблица:
          - "Замечание"
          - "Вероятность (САВ)" (0–1 / %)
          - "Алгоритм/параметры обработки"
          - "Источник решения" (МР/ДМР/ИИМ-vX — при необходимости)
      - Вкладка "Решения":
        - Таблица:
          - "Замечание"
          - "Решение" (чип: ДА / НЕТ / Н.О.)
          - "Источник решения" (МР / ДМР / ИИМ-vX)
          - "Эталон" (значение, если есть)
          - "Комментарий эксперта" (иконка/текст)

8. Экран "Набор" (детальная страница)
- Frame: "Desktop / Набор — детали"
  - Topbar + Navbar: активен таб "Наборы"
  - Верхняя часть:
    - Breadcrumb: "Наборы / <Название набора>"
    - Page Header:

        - Secondary button: "Редактировать"
  - Макет:

      - Card "Основная информация":

          - "Автор"
      - Card "Статистика" (опционально):
        - "Кол-во осмотров"
        - "Последний запуск" (дата, статус)
    - Нижний блок:
      - Card с заголовком "Осмотры в наборе"
      - Верхняя панель:
        - Search input: "Поиск по осмотрам"
        - Button (secondary): "Добавить осмотры"
        - Button (ghost/danger): "Удалить выбранные"
      - Таблица "Осмотры набора":
        - Колонки:
          - [Checkbox]

          - "Дата осмотра"

          - "Источник"

9. Итог
- Все перечисленные фреймы разместить на странице "Desktop" в Figma, выровнять по вертикальной сетке.
- Общие компоненты (Topbar, Navbar, Button, Table, Card, Tag, Tabs, Pagination) оформить как отдельные компоненты на отдельном участке страницы или на отдельной странице "Library".
- Цвета и типографику вынести в Styles (Color Styles, Text Styles) для дальнейшего масштабирования.
