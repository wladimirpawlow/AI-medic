# Общие данные
Экран должен состоять из трех секций:
- общие данные по осмотру
- материалы (справа от общих данных)
- перечень запусков по осмотру
- замечания

Секции расположены в визуально отделенных блоках, которые нельзя свернуть.

# Общие данные по осмотру
Данная секция должна содержать следующую информацию:
- внутренний id медосмотра
- внешний идентификатор медосмотра
- идентификатор контрагента
- идентификатор сотрудника
- идентификатор организации 
- идентификатор МР
- идентификатор мед-организации
- идентификатор ПАК
- тип ПАК
- тип осмотра

- условия проведения (температура, влажность, освещенность)
- результаты измерений (алкоголь, давление, температура)

Данные параметр-значение должны быть выведены один по другим.

# Материалы
Данная секция должна содержать следующие элементы:
- видеозапись
- таймкоды видеозаписи с указанием времени ключевых событий видеозаписи (секунды от начала записи) (располагаются под видеозаписью)
- фото сотрудника, проходившего медосмотр
- набор кадров высокого разрешения с таймкодами привязки ко времени видеозаписи (миллисекунды от начала записи) (располагаются под видеозаписью)

# Перечень запусков по осмотру
Данная секция должна содержать следующие элементы:
- таблицу запусков по данному осмотру
- кнопку "запустить анализ САВ"

## описание таблицы
| номер столбца | имя столбца | описание | тип данных |
|-|-|-|-|
|1| id запуска | идентификатор запуска | UUID |
|2| версия САВ | версия САВ, которыя использовалась для запуска | UUID |
|3| id набора | идентификатор набора по которому проводится запуск и номер версии (пусто, если запуск проводился по единичному осмотру) | UUID & INT | 
|4| статус | текущий статус (завершен успешно, остановлен, в очереди, обработка) , количество завершенных (обработанных) осмотров  | N/M + инфографика|
|5| рассчитать | запуск рассчета. Доступен только для завершенных запусков  | кнопка/иконка рассчета|

## описание кнопки
При нажатии открывается модальное окно с выпадающим списком версий САВ и кнопками "запустить"/"отменить"

# замечания
Данная секция должна содержать 
- таблицу замечаний

## описание таблицы
1. Таблица должна содержать перечень всех активных замечаний в системе аналогично экрану "замечания" с дополнительной информацией по решениям и возможностью редактирования эталонного решения. 
2. Структура таблицы:
| номер столбца | имя столбца | описание | тип данных |
|-|-|-|-|
|1| группа | данные аналогично экрану "замечания" | текст |
|2| наименование | данные аналогично экрану "замечания" | текст |
|3| эталон |  да/нет/- (прочерк если эталон не задан)| да/нет/-|
|4| решение МР | да/нет и информация о несовпадении с эталоном |  да/нет & инфографика | 
|5| решение ИИ-медик | да/нет и информация о несовпадении с эталоном |  да/нет & инфографика |
|5| результаты анализа | таймкоды и вероятности наличия замечания на видео |  массив данных в формате: [(0,9 - 0..35c), (0,8 - 35..55c)..] |

3. В шапке табличе в столбце "эталон" рядом с заголовком должна быть иконка "редактировать".
При нажатии данные в столбце да/нет/- меняются на чербоксы. Иконка редактировать меняется на "сохнатнить".
При нажатии на сохранить записываются обновленные данные эталона в соответствии с чекбоксами.

4. Данные в столбцах "решение ИИ-медик", "результаты анализа" появляются после проведения расчета по запуску. 

Кроме того после проведения расчета:
- Стока расчета в таблице запусков остается выделенной до момента запуска расчета по другому запуску. Когда проводится рассчет по другому запуску выделение гаснет
- напротив иконки рассчитать появляется ИД алгоритма по которому был реализован рассчет. Когда проводится рассчет по другому запуску ИД алгоритма исчезает
