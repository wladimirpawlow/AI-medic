# Общие данные
На экране должна отображаться таблица с полным перечнем всех запусков, которые были реализованы в систему.

Описание таблицы :
| номер столбца | имя столбца | описание | тип данных |
|-|-|-|-|
|1| id запуска набора | идентификатор запуска | UUID |
|2| версия САВ | версия САВ, которыя использовалась для запуска | UUID |
|3| id набора | идентификатор набора по которому проводится запуск и номер версии | UUID & INT | 
|4| имя осмотра | наименование набора по которому проводится запуск | UUID | 
|5| инициатор | логин инициатора старта запуска | строка |
|6| старт | дата/время начала запуска | дата/время | 
|7| завершение | дата/время завершения обработки запуска | дата/время | 
|8| статус | текущий статус (завершен успешно, остановлен, в очереди, обработка) , количество завершенных (обработанных) осмотров  | N/M + инфографика|
|9| рассчитать | запуск рассчета. Доступен только для завершенных запусков  | кнопка/иконка рассчета|

Для каждого незавершенного запуска должна быть предусмотрена возможность остановить процесс обработки. Функционал должен быть реализован через кнопку на нфографике в колонке "статус" в стиле play/stop. Кнопка доступна только для строк со статусом  "в очереди", "обработка". После нажатия кнопки перед остановкой должно открыться модальное окно с подтверждением действия.


Должна быть реализована возможность сортировки полей таблицы по столбам "старт" и "завершение" по возрастанию и убыванию.

Должна быть реализована возможность запустить рассчет для всех завершенных запусков. Функция должна выть реализована в виде кнопки в дополнительном столбце. При нажатии кнопки открывается модальное онко с данными:
- выбор алгоритма и выпадающий список доступных алгоритмов
- кнопки "рассчитать" и "отменить"
