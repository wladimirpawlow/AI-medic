# Общие данные
На экране должна отображаться таблица с полным перечнем всех осмотров, которые были загружены в систему.

Описание таблицы :
| номер столбца | имя столбца | описание | тип данных |
|-|-|-|-|
|1| id | внутренний идентификатор медосмотра | UUID |
|2| внешний id | внешний идентификатор медосмотра | UUID |
|3| контрагент | идентификатор контрагента | UUID или строка |
|4| id сотрудника | идентификатор сотрудника | UUID |
|5| id ПАК | идентификатор ПАК | UUID |
|6| тип ПАК | тип ПАК | ENUM (стацонарный, мобильный) |
|7| эталон | наличие эталона | инфографика для ENUM (нет эталона, совпаает с МР, не совпадает с МР ) |
|8| замечания МР | наличие замечаний по медосмотру от МР | иконки замечаний |
|9| дата загрузки | дата загрузки медосмтора | дата-время |


При нажатии на id должен осуществляется переход на экран данного осмотра.
В каждой строке должен быть чекбокс.
В шапке таблице должен быть элемент выделить/снять все чекбоксы по всем строкам в рамках отфильтрованных строк.

Должна быть реализована возможность сортировки полей таблицы по столбцу "дата загрузки" по возрастанию и убыванию

# Добавление в наборы
На экране должен быть реалзован элемент (кнопка) "добавить выбранное в набор".
При нажатии на кнопку должно открываться модальное окно со следуюшими функциями:
- выбор доступных наборов из выпадающего списка
- кнопка "добавить"
- кнопка "отменить"

# Загрузка осмотров
На экране должен быть реалзован элемент (кнопка) "загрузить осмотры".
При нажатии на кнопку должно открываться модальное окно со следуюшими функциями:
- текстовое поле для ввода массива внешних идентификаторов 
- кнопка "загрузить"
- кнопка "отменить"
