# Ролевая модель
Приложение предназначено для работы следующих групп пользователей:
- администратор
- эксперт
- пользователь

# Структура интерфейса.
Интерфейс должен содержать следующие основные экраны:
- осмотры - таблица всех загруженных осмотров 
- наборы - таблица всех сформированных наборов 
- запуски - таблица всех запусков 
- расчеты - таблица всех сформированных расчетов 
- замечания - таблица замечаний в с которыми работает система

Интерфейс должен содержать следующие дополнительные экраны:
осмотр - детальная страница осмотра с возможностью просмотра деталей осмотра
набор - детальная страница набора с возможностью просмотра всех осмотров в рамках набора и корректировки перечня
запуск - детальная страница запуска с перечнем всех осмотров в запуске и их статусом


# Статическое меню
Интерфейс должен содержать верхнее статическое меню.
Переход на основные экраны должен быть доступен из верхнего статического меню.
Переход на дополнительные экраны должен осуществляться через элементы основных экранов.

В рамках статического меню должен быть блок "пользователь" с отображением логина и роли текущего пользователя, а также функцией выхода из системы.

# Общие требования.
Дизайн должен бть реализован в минималистичном стиле для корпоративных систем.
Предпочтительны светлые серые, голубые и синие цвета.

# Требования к таповым элементам
## Таблицы 
- Все таблицы в системе должны иметь встроенные фильтры по каждому столбцу кропе столбцов с датами и временем.

- Для столбцов с датами и временем должны быть реализованы фильтры над таблицей. Фильтры должны быть реализованы в выделенном блоке "фильтры". Блок фильтров должен быть сорачиваемым. Параметры вильтрации располагаются один под другим в виде строк: параметр - даипазон значений.
- Все таблицы в системе должны содержать возможность пагинации с отображением 10/20/50/100 элементов.

## Модальные окна
Модальные окна должны открываеться поверх текущего экрана.
Модальные окна должны располагаться по центру экрана.
